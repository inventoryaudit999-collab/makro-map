# MAKRO MAP — แผนที่สาขาแม็คโครทั่วประเทศ

เว็บแอปแผนที่สาขาแม็คโคร (CP AXTRA) 166 สาขา พร้อมระบบล็อกอิน, วัดระยะทางระหว่างสาขา, บันทึก/แก้ไข/ลบ ระยะทาง, Export Excel, สลับภาษา (ไทย / EN / 한국), และซิงค์ข้อมูลข้ามอุปกรณ์ผ่าน Firebase

ไฟล์ทั้งหมดอยู่ใน **`index.html` ไฟล์เดียว** เปิดใช้งานได้ทันที

---

## วิธีใช้งานเบื้องต้น

เปิด `index.html` ด้วยเบราว์เซอร์ได้เลย (ดับเบิลคลิก)
- **ล็อกอิน:** ผู้ใช้ `admin` / รหัส `admin@1234`
- ทำงานได้ทันทีแบบออฟไลน์ (บันทึกระยะทางเก็บในเครื่อง)
- ต่อ Firebase เมื่อต้องการให้หลายอุปกรณ์เห็นข้อมูลชุดเดียวกัน

---

## ติดตั้งบน GitHub Pages (ฟรี เข้าจากทุกอุปกรณ์)

ขั้นตอนน้อยที่สุด — แค่ 4 ขั้น:

1. สร้าง repository ใหม่บน GitHub (เช่นชื่อ `makro-map`)
2. กดปุ่ม **Add file → Upload files** แล้วลากไฟล์ `index.html` และ `.nojekyll` เข้าไป กด **Commit**
3. ไปที่ **Settings → Pages** → ในหัวข้อ *Source* เลือก **Branch: main** / โฟลเดอร์ **/(root)** → กด **Save**
4. รอ ~1 นาที จะได้ลิงก์ `https://<ชื่อผู้ใช้>.github.io/makro-map/` เปิดได้จากมือถือ/แท็บเล็ต/PC

> ไฟล์ `.nojekyll` มีไว้กันปัญหา GitHub Pages ซ่อนไฟล์ ไม่ต้องแก้อะไร

## ต่อ Firebase (ตั้งค่าให้แล้ว — พร้อมใช้ทันที)

ไฟล์นี้ **ใส่ค่า Firebase ของโปรเจกต์ `makro-map` ไว้เรียบร้อยแล้ว** อัปโหลดขึ้น GitHub ได้เลย ข้อมูลจะซิงค์ realtime ข้ามทุกอุปกรณ์โดยอัตโนมัติ

**เหลือเพียงขั้นตอนเดียว — ตั้งค่า Rules ของ Realtime Database** (ครั้งเดียว) เพื่อให้แอปอ่าน/เขียนข้อมูลได้:

1. ไปที่ https://console.firebase.google.com → เลือกโปรเจกต์ **makro-map**
2. เมนูซ้าย **Build → Realtime Database → แท็บ Rules**
3. วางกฎด้านล่างนี้ แล้วกด **Publish**

```json
{
  "rules": {
    "makro_branches": { ".read": true, ".write": true },
    "makro_distance_logs": { ".read": true, ".write": true }
  }
}
```

ครั้งแรกที่เปิดแอป ระบบจะ seed ข้อมูล 166 สาขาขึ้น Database อัตโนมัติ มุมขวาบนจะขึ้น **ซิงค์ออนไลน์** สีเขียว จากนั้นบันทึกระยะทางทุกครั้งจะเก็บบน Firebase และทุกอุปกรณ์เห็นพร้อมกันแบบ realtime

> **หมายเหตุความปลอดภัย:** กฎข้างต้นเปิดให้อ่าน/เขียนแบบสาธารณะ เหมาะกับการใช้งานภายในทีม หากต้องการจำกัดสิทธิ์ที่เข้มขึ้น (เช่นต้องยืนยันตัวตนก่อนเขียน) สามารถปรับ Rules เพิ่มเติมได้ภายหลัง

## หมายเหตุเรื่องข้อมูล

- ชื่อสาขา รหัส (Loc) พิกัด LATITUDE/LONGITUDE และที่อยู่ อ้างอิงจากไฟล์ `MAKRO_ALL.xlsx` (พิกัดจริง แม่นยำระดับหน้าร้าน)
- ชื่อภาษาไทยอ้างอิงจาก `ALL_STORE.xlsx` ชื่ออังกฤษจากคอลัมน์ Loc Name ชื่อเกาหลีเป็นการถอดเสียง
- หากต้องการปรับข้อมูลสาขา แก้ที่ Realtime Database (โหนด `makro_branches`) ได้โดยตรง ทุกอุปกรณ์อัปเดตตาม
- ระยะทางที่วัดเป็น **เส้นตรง (haversine)** เหมาะกับการเปรียบเทียบ/วางแผน ส่วนระยะถนนจริงให้กดปุ่มนำทาง Google Maps
